## 4. 검증 방법론 평가

### R11 v70의 혁신: 예상-실측 대조

기존 검증과의 차이점:
- **기존**: 실행 후 결과만 보고, 사후 분석
- **v70**: 시나리오 코드에 **예상 분기를 사전 기록** → 가설 검증 방식 도입

```python
# 코드 예시 (scenarios_reliability_v70.py)
("G2_sc01", [
  ("미국 관광비자로 입국 후 취업으로 바꿀 수 있나요?",
   EXC,  # ← 실행 전에 기록한 예상 분기
   "관광→취업 신분변경 → exception_handler")
], ...)
```

이 방식 덕분에 단순히 "틀렸다"가 아닌, **왜 예상과 달랐는지** 구조적으로 분석 가능.

| 우선순위 | 영역 | 현재 | 원인 | 해결 방향 |
