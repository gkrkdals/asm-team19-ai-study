## Round 11 — 신뢰도 종합 검증(예상–실측 대조)·분석 시각화·UI 개선

### 프론트엔드
- **상단 고정(pin) 고도화**: 대화 목록에 📌 표시(좌상단 배지 + 우상단 코너) + ⋮ 메뉴에서
  **고정/해제 토글**. 고정 상태는 백엔드(`pinned`)에 영속, 목록은 **고정 우선 → 생성시각
  내림차순**으로 정렬.
- **인사말 하단 추천 질문**: 새 대화 화면에서 추천 질문 풀(15개) 중 **랜덤 4개**를 칩으로
  노출, `🎲 다른 질문`으로 즉시 재추첨. 클릭 시 바로 전송.
- **세션 관리 알림**: 이름변경·삭제·고정·세션전환 시 **토스트 알림**, 삭제는 **확인 모달**로
  사용자 의도 재확인.

### 검증·분석
- **신뢰도 종합 검증 v70(예상–실측 대조)**: 70개 시나리오(A/B/C 기본 20 + 예외·특수 20 +
  후속질문 연계 30×10턴)를 **예상 분기를 코드에 사전 기록**한 뒤 실제 라우팅과 대조.
  (`tests/scenarios_reliability_v70.py`, `tests/run_reliability_v70.py`)
- **신뢰도 분석 페이지(`/analytics`)**: 라운드별 전체율·저신뢰영역(신분변경/멀티턴/충돌)
  개선 추이를 **표·그래프(Chart.js)** 로 시각화. `tests/build_analytics.py` 가 각 라운드
  결과 JSON을 모아 `web/public/analytics_data.json` 으로 집계.

### 버그 수정
- **세션 격리**: 스트리밍 도중 새 세션 생성 시 응답이 엉뚱한 세션에 기록되던 문제 수정
  (전송 시점 `sentSid` 캡처).
- **프론트엔드 먹통(2중 버그)**: ① `<button>` 안 `<button>`(HTML 명세 위반)로 DOM 붕괴 →
  `.conv`를 `div[role=button]`, `.conv-more`를 `span`으로 변경. ② `$('id')`를 `#` 없이 호출해
  `null.addEventListener` TypeError로 IIFE 초기화 중단 → `$('#id')`로 수정.
- **응답 품질**: "처리 기간·수수료" 섹션을 검색결과에 해당 정보가 있을 때만 표기(빈도 감소),
  **존재하지 않는 URL 생성 차단**(검색결과의 실제 URL만 인용 → DNS_PROBE_FINISHED_NXDOMAIN 방지).
- **트레이스 END 노드**: 대화 완료 시 종료 노드를 **녹색(done)** 으로 표시.

## Round 10 — 멀티턴 신뢰도·ChromaDB 자가학습 증명

- **멀티턴 부정신호 처리 수정(핵심)**: `intent.py`의 부정/충돌 신호 처리를 **‘맥락 전체 폐기’
  → ‘국가 보존 + 목적만 교체’** 로 변경. 깊은 대화(5턴+)에서 확정 정보를 명시적으로 유지.
  → 멀티턴/맥락전환/충돌 영역이 **33%/0% → 100%(238턴)** 로 회복.
- **ChromaDB 자가학습 라이브 증명(4/4)**: DB 0건 신규국가(IS/LU/EE/UY)로 BEFORE→[웹검색
  +`knowledge_writer`]→AFTER(+1)→재요청[RAG만, 웹검색 없음] 전체 사이클 입증.
- **Streamlit → FastAPI SPA → Node.js(Express) 분리**: 고객 UI를 `:3000`(Express, 정적 서빙),
  API를 `:8000`(FastAPI)로 단일화.

## 추가 개선(5차) — AI 후속칩·반응형 트레이스 레이아웃·종합 검증

- **상황별 AI 후속 질문 칩**: 답변 직후 `POST /chat/followups` 로 대화 맥락(국가·목적·언급 비자)을
  반영한 후속 질문 4개를 **LLM 이 동적으로 생성**해 칩으로 노출. 세션 `last_run` 에 저장돼
  새로고침에도 보존되며, 실패 시 정적 폴백.
- **트레이스 그래프 반응형 배치 + 사용자 줌**: 공간이 충분하면 가로(LR), 좁으면 세로(TB)로 자동
  배치(수동 토글 가능). 줌(−/＋/맞춤/100%)으로 좁은 화면에서도 모든 노드를 한눈에.
- **종합 검증(시나리오 A/B/C + 전체 노드)**: 9개 노드 전부 도달 가능성 확인(query_refiner 는
  웹검색 신뢰도 'poor' 시 동작 — 평시 미발동, 라우팅/엣지로 도달성 확인).

## 추가 개선(4차) — 세션 영속·후속요청·세션별/통합 트레이스

- **선택 노드/간선 강조**: 실행 경로 노드는 강한 글로우 + 다크 헤일로 + 약간 확대로 최상위
  z-index 렌더.
- **후속 요청 제안(ChatGPT 식)**: 답변 직후 `더 구체적으로 / 최신 정보 / 다른 비자 종류 / 공식
  사이트 상세 탐색` 버튼. 클릭 시 사용자가 입력한 것처럼 자동 실행(멀티턴 맥락 활용).
- **공식 사이트 상세 탐색(딥서치)**: '상세 탐색/공식 사이트/최신 정보' 요청 시 RAG 를 건너뛰고
  `web_search_tool` 직행 + Tavily `search_depth=advanced` + 원문(raw_content)으로 깊게 탐색.
- **대화 영속화(새로고침 보존)**: 백엔드 RESTful 세션 스토어(`/sessions`, `data/sessions.json`).
  활성 세션은 URL `?sid=` 로 유지.
- **세션 메타데이터**: 이름 변경·한줄 설명·태그(장기체류/취업/유학/여행…) 편집·표시.
- **세션별 RESTful 트레이스(2a)** + **통합 병렬 허브(2b)**: 세션은 `?sid=<id>` ↔ `/{id}/trace`
  1:1 매핑. `/trace` 는 모든 세션을 카드로 한눈에.

## 추가 개선(3차)

- **데스크탑 트레이스 레이아웃**: 단계별 파이프라인을 전체 폭 반응형 카드 그리드로, 모바일은
  단일 컬럼.
- **지식 자가 학습(knowledge_writer)**: 웹 검색 결과가 신뢰도 **good** 이면 ChromaDB 에 학습
  문서로 upsert(`origin=web_search`). 다음에 같은 국가는 RAG 즉답.
- **예외 시나리오 보강(13→21종)**: I-539 체류연장, 신분변경 제한, 긴급/급행 비자, 오버스테이
  입국금지(3/10년 바), 유학생 근로시간, 여권 6개월 유효, DUI 입국거부, 디지털 노마드 등.
- **3-pane 프론트엔드** + **워크플로우 패널 열기/닫기** + **의도추출 키워드 폴백** +
  **트레이스 그래프 가독성(z-index 레이어링)**.

## 추가 개선(2차)

- **의도 분류 강건화**: `intent_classifier`를 결정적(temperature=0)으로 실행, 도메인 키워드·
  국가/목적 신호가 있으면 LLM 오판을 무시하고 비자 플로로 강제.
- **멀티 채팅 세션**: 사이드바 ➕ 새 대화 생성·전환·삭제, 대화별 맥락 유지.
- **Tavily 검색어 정제**: 한국어·`Korea` 배제, 핵심 영어 쿼리(국가명 + visa + requirements/
  eligibility/official)만 사용.

## 데모 리뷰 반영(개선 이력)

- **장기 체류 비중 강화**: 단기→영주권·정착 경로와 갱신/전환 조건 함께 안내.
- **교차 예외규칙 보강**: 쉥겐↔비쉥겐, 환승, ESTA/eTA, 유효기간≠체류 등 13종 규칙을 RAG 적재.
- **전세계 국가 대응**: 6개국은 RAG, 그 외는 `search_hints` 의 우선 공식도메인·검색어 템플릿.
- **검색 신뢰도 게이트 + 검색어 재생성 루프**: 공식 출처 부족 시 LLM이 영어 검색어 재생성.
- **일반 대화 분기**: 비자 무관 질문을 `general_chat` 로 분리.
- **멀티턴 맥락 / 응답 토큰 스트리밍 / SPA ↔ /trace 실시간 연동**.
